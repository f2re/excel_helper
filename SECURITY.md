# 🔒 Политика безопасности

## Поддерживаемые версии

На этапе 0.x исправления безопасности выпускаются для последней версии ветки `main`.

## Сообщение об уязвимости

Не создавайте публичный issue, если проблема может раскрыть:

- персональные данные;
- путь к внутреннему хостингу;
- закрытые ключи;
- возможность выполнить произвольный код;
- способ обойти политики Office/организации.

Используйте приватный канал владельца репозитория или GitHub Security Advisories, если он включён.

В сообщении укажите версию, платформу Excel, манифест, шаги воспроизведения и минимальный обезличенный пример.

## Секреты

Никогда не коммитьте:

```text
.certs/
.env
production private keys
книги с реальными кадровыми данными
```
