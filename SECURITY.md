# 🔒 Security Policy

Не публикуйте персональные данные, реальные групповые расписания или кадровые книги в issue.

Для сообщения об уязвимости используйте приватный канал владельца репозитория GitHub. Укажите версию, клиент Excel, воспроизводимый сценарий, влияние и минимальный обезличенный пример.

Проект не считает скрытые листы средством защиты от пользователя, имеющего доступ к книге.
