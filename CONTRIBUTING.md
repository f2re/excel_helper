# 🤝 Участие в разработке

## Перед изменением

1. Откройте issue с воспроизводимым сценарием.
2. Не прикладывайте реальные персональные данные.
3. Для ошибок расписания подготовьте обезличенную копию минимального листа.

## Проверки

```bash
npm ci
npm run check
```

Изменение считается готовым, если:

- добавлены тесты чистой логики;
- документация описывает новый сценарий;
- существующие таблицы мигрируют идемпотентно;
- новый Office API имеет проверку requirement set или fallback;
- пользовательские данные не удаляются без явного подтверждения.

## Pull request

В описании укажите:

- проблему и пользовательский сценарий;
- изменённые листы/таблицы;
- изменения UI;
- совместимость;
- тесты;
- screenshots без персональных данных, если изменён интерфейс.
