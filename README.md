# 🧰 ПрофиПомощник для Excel

**ПрофиПомощник** — русскоязычная надстройка Microsoft Excel для кадровых служб, рекрутинга, преподавателей, учебных подразделений и повседневного планирования.

В одном пакете:

- **114 пользовательских функций** в ячейках с русскими именами;
- **44 апплета** для массовых операций, шаблонов и отчётов;
- **Excel-native составитель сводного расписания**, перенесённый из фактического потока [`f2re/planer-solving`](https://github.com/f2re/planer-solving);
- боковая панель с поиском, справкой и вставкой формул;
- вкладка ленты Excel с основными командами;
- автоматическое создание и восстановление всех служебных листов и таблиц.

> Надстройка не требует отдельного Python-сервиса, CLI или базы данных. Исходные групповые листы, промежуточные таблицы, назначения, конфликты и версии результата хранятся в текущей книге Excel.

## ✨ Что входит

| Направление | Возможности |
|---|---|
| 👥 Кадры | ФИО, инициалы, стаж, возраст, должности, ставки, договоры, телефоны, email, ИНН, СНИЛС |
| 🎯 Рекрутинг | Дубли кандидатов, SLA, навыки, опыт, зарплата, рейтинг, нагрузка рекрутера, стоимость найма |
| 🎓 Обучение | Количество и часы занятий, академические часы, конфликты преподавателей, групп и аудиторий, посещаемость, оценки |
| ✅ Планирование | Статусы задач, дедлайны, просрочки, загрузка, смены, план-факт, недельные планы, Канбан и Гант |
| 🧹 Данные | Нормализация текста, раскладка, транслитерация, даты и числа из текста, дубли, похожесть, списки, регулярные выражения |
| 📅 Сводное расписание | Импорт групповых листов, ручная корректировка парсера, легенда преподавателей, потоки, Round-Robin, аудит и итоговые листы |

## 🚀 Быстрый старт разработчика

Требования: Node.js 20+ и настольный Excel Microsoft 365 либо Excel в браузере.

```bash
git clone https://github.com/f2re/excel_helper.git
cd excel_helper
npm ci
npm run check
npm start
```

После запуска откройте Excel и загрузите `manifest.xml` или используйте команды отладки, описанные в [руководстве по установке](docs/INSTALL.md).

## 📅 Как работает сводное расписание

```mermaid
flowchart LR
    A[Чистая книга Excel] --> B[Составить сводное расписание]
    B --> C[Автогенерация служебных листов]
    C --> D[Импорт или выбор группового листа]
    D --> E[Группа вручную или из выбранной ячейки]
    E --> F[Автопоиск структуры]
    F --> G{Парсер угадал?}
    G -- Нет --> H[Исправить строки, столбцы и смещения]
    H --> F
    G -- Да --> I[Сохранить источник]
    I --> J[Сопоставить преподавателей]
    J --> K[Потоки и Round-Robin]
    K --> L[Аудит конфликтов]
    L --> M[Сводный, недельный и личные листы]
```

При первом нажатии **«Составить сводное расписание»** мастер:

- создаёт отсутствующие системные листы и именованные таблицы;
- не удаляет данные из уже существующих таблиц;
- предлагает импортировать `.xlsx` или выбрать лист текущей книги;
- позволяет выбрать группу вручную или закрепить адрес ячейки, в которой она находится;
- показывает все найденные координаты парсера до сохранения;
- позволяет изменить каждый параметр, если структура определена неверно.

Подробный поток, правила потоковых занятий, Round-Robin, ручные закрепления, конфликты и обновление источников описаны в [📅 SCHEDULE_WORKFLOW.md](docs/SCHEDULE_WORKFLOW.md).

## 🗂 Служебная структура

Служебные элементы создаются автоматически и обычно скрыты (`veryHidden`).

| Лист | Основные таблицы | Назначение |
|---|---|---|
| `_PROFI_SETTINGS` | `tblScheduleSettings`, `tblParserProfiles`, `tblProfiStatuses` | Настройки семестра, профили парсера, статусы |
| `_PROFI_TEACHERS` | `tblTeachers`, `tblTeacherAliases` | Преподаватели и варианты написания ФИО |
| `_PROFI_SOURCES` | `tblSources` | Подключённые групповые листы |
| `_PROFI_DATA` | `tblLessons`, `tblTeacherCandidates`, `tblCalendar` | Нормализованные занятия, кандидаты и календарь |
| `_PROFI_CALC` | `tblAssignments`, `tblOccupancy`, `tblConflicts`, `tblManualOverrides` | Назначения, занятость, конфликты, ручные правила |
| `_PROFI_LOG` | `tblOperationLog`, `tblScheduleVersions` | Журнал и снимки результата |

Если пользователь удалил лист или таблицу, повторное открытие мастера либо команда **«Проверить структуру книги»** восстановит недостающие элементы.

## 📊 Итоговые листы расписания

| Лист | Содержание |
|---|---|
| `Сводное расписание` | Преподаватели × пары × даты семестра |
| `Неделя` | Интерактивная выбранная неделя |
| `По преподавателю` | Личное расписание выбранного преподавателя |
| `Контроль расписания` | Конфликты, предупреждения и LessonID |
| `Нагрузка преподавателей` | Недельные и суммарные показатели |
| `Занятость аудиторий` | Использование помещений |
| `Печать Нxx` | Печатные листы, создаваемые по запросу |

## 🧠 Порядок назначения преподавателей

1. ручное закрепление;
2. сохранение допустимого назначения из предыдущей версии;
3. потоковое занятие той же дисциплины;
4. единственный кандидат;
5. детерминированный Round-Robin;
6. неназначенное занятие либо совместимый конфликт — в зависимости от настройки.

Исходные групповые листы не переписываются.

## 🔐 Приватность

- данные не отправляются в сторонние сервисы;
- групповые расписания вставляются в текущую книгу;
- расчёты выполняются в runtime надстройки;
- исходные листы скрываются, но не изменяются;
- экспорт `.ics` создаётся локально в браузерном контексте панели.

Подробнее: [SECURITY_AND_PRIVACY.md](docs/SECURITY_AND_PRIVACY.md).
