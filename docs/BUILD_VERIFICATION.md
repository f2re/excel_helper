## 🧹 Проверка чистой среды

Проверка выполнялась после удаления локальных каталогов сборки и зависимостей:

```bash
rm -rf node_modules dist .test-dist
npm ci
npm run check
```

Команда `npm run check` последовательно выполняет генерацию справочников, проверку TypeScript, тесты, проверку документации, production-сборку, локальную валидацию манифеста и smoke-проверку готового `dist/`.

## ⚠️ Ручная приёмка в Excel

Автоматическая проверка не заменяет запуск в реальном клиенте Excel. Перед производственным развёртыванием рекомендуется выполнить сценарии из [`TESTING.md`](TESTING.md):

1. sideload манифеста в настольный Excel Microsoft 365;
2. открытие панели и проверка поиска;
3. запуск функции в ячейке;
4. создание служебной структуры в чистой книге;
5. импорт тестового группового расписания;
6. ручная коррекция профиля парсера;
7. формирование сводного, недельного и личного расписаний;
8. закрытие и повторное открытие книги;
9. проверка Excel в браузере и на macOS при наличии этих целевых платформ.

Официальная сетевая проверка манифеста Microsoft требует доступа к сервисам Microsoft и запускается отдельно командой:

```bash
npm run validate:office
```
