# 🧯 Диагностика

## `npm start` открыл непонятную пустую страницу

Актуальная версия должна открывать dashboard:

```text
https://localhost:3000/
```

На нём должны быть версия, число функций, число апплетов и ссылки на манифесты. Task pane preview находится по адресу:

```text
https://localhost:3000/src/ui/taskpane.html
```

Если страница старая:

```bash
rm -rf dist
npm run build:dev
npm start
```

Проверьте, что браузер не использует кэш и что `https://localhost:3000/health` возвращает JSON.

## Browser preview показывает «Требуется Excel»

Это штатное поведение. В обычном браузере работают поиск, каталог и стенд функций. `Excel.run` доступен только внутри Excel после загрузки манифеста.

## Вкладки task pane пустые

1. Откройте консоль браузера.
2. Проверьте загрузку `src/runtime.js` и `src/ui/taskpane.js`.
3. Откройте `https://localhost:3000/health`.
4. Выполните `npm run syntax`.
5. Выполните `npm run build:dev` повторно.

Панель должна инициализироваться по `DOMContentLoaded`, даже если Office.js недоступен.

## `functions.json` не появился

`npm run build`, `npm run build:dev` и `npm start` запускают генератор автоматически.

Проверьте вручную:

```bash
npm run generate
ls -l src/functions.json dist/functions.json
```

Затем:

```bash
npm run validate
npm run smoke
```

## Изменения функции не попадают в browser preview

При работающем `npm start` watch следит за `src/`, `assets/` и манифестами. В терминале должна появиться строка о пересборке.

Если watch недоступен:

```bash
npm run build:dev
```

и обновите страницу вручную.

## Сертификат не доверен

### macOS

```bash
npm run certs:trust:mac
```

Полностью перезапустите браузер и Excel.

### Другие системы

Создайте сертификат:

```bash
npm run certs
```

Импортируйте `.certs/cert.pem` в доверенные корневые сертификаты пользователя.

## OpenSSL не найден

На macOS:

```bash
xcode-select --install
```

или:

```bash
brew install openssl
```

Альтернатива:

```bash
npx --yes office-addin-dev-certs@2.0.10 install
```

## Порт 3000 занят

Найдите процесс:

```bash
lsof -i :3000
```

Остановите его либо запустите сервер вручную на другом порту:

```bash
npm run build:dev
npm run certs
node scripts/dev-server.mjs --port=3001 --open
```

Для другого порта нужно также пересобрать манифест с соответствующим `--base-url`.

## Надстройка не появилась в Excel Mac

1. Выполните `npm run sideload:mac`.
2. Проверьте файл в `~/Library/Containers/com.microsoft.Excel/Data/Documents/wef/`.
3. Полностью закройте Excel.
4. Запустите `npm run start:no-open`.
5. Убедитесь, что `/health` открывается без предупреждения.
6. Запустите Excel заново.

Удаление и повторная установка:

```bash
npm run unsideload:mac
npm run sideload:mac
```

## Панель Office.js не открывается в Excel

- проверьте HTTPS и доверие сертификату;
- убедитесь, что URL из манифеста доступен;
- проверьте соответствие манифеста клиенту;
- для Excel 2016 используйте `manifest-office2016.xml`;
- при отсутствии requirement set используйте XLAM;
- очистите кэш Office после изменения манифеста.

## `#NAME?` для `PROFI.*`

Причины:

- Custom Functions не поддерживаются данным клиентом;
- загружен Excel 2016 task-pane-only манифест;
- `functions.json` не доступен по URL манифеста;
- Excel сохранил старый манифест в кэше.

Проверьте:

```text
https://localhost:3000/functions.json
```

В старом Excel используйте VBA-функции `PROFI_*` из XLAM.

## `npm run check` проходит, но Excel не работает

Автоматические тесты не заменяют интеграционную приёмку в реальном Excel. Проверьте:

1. sideload манифеста;
2. доверие сертификату;
3. статус среды в task pane;
4. вызов функции в ячейке;
5. создание служебной схемы;
6. импорт реального группового листа;
7. повторное открытие книги.

## PowerShell не собирает XLAM/XLTM

- сборка выполняется только в Windows;
- Excel должен быть установлен;
- включите доступ к объектной модели VBA-проекта;
- проверьте, что антивирус не блокирует COM Automation;
- выполните `npm run legacy:prereq`;
- изучите [MACOS.md](MACOS.md), если разработка идёт на Mac.

## Можно ли собрать XLAM на Mac через openpyxl или XlsxWriter

Эти библиотеки могут сохранить или внедрить уже готовый `vbaProject.bin`, но не компилируют `.bas` в VBA-проект. Для воспроизводимой бинарной сборки нужен Windows Excel runner либо заранее собранный seed.

## Группа определилась неверно

Используйте:

- ручной ввод;
- имя листа;
- выбранную ячейку.

В `tblSources` можно изменить режим и адрес. Значение из ячейки читается повторно при обновлении.

## Парсер не нашёл недели или легенду

Покажите служебные листы, откройте `tblParserProfiles`, исправьте:

- `WeeksRow`;
- `WeekStartCol`;
- `GridStartRow`;
- `LegendRow`;
- `RowsPerDay`;
- `RowsPerPair`;
- смещения кода, дисциплины и аудитории;
- столбцы легенды.

Затем запустите расчёт повторно.

## Все преподаватели не назначены

1. Заполните `tblTeachers`.
2. Добавьте варианты написания в `tblTeacherAliases`.
3. Проверьте столбцы легенды.
4. Проверьте признак `Активен`.
5. Проверьте строгий режим конфликтов.

## Одинаковые фамилии назначаются неверно

Не используйте фамилию без инициалов как универсальный алиас. Создайте точные записи `Иванов И.И.` и `Иванов П.С.` с разными TeacherID.

## После обновления расписание сильно изменилось

Проверьте:

- сохранение прежних допустимых назначений;
- ручные закрепления;
- стабильность SourceID;
- изменение группы или профиля парсера;
- активность преподавателей;
- удалённые алиасы.

Перед пересчётом используйте снимок версии и сравнение изменений.
