# 🚢 Выпуск версии

## 1. Подготовка

- [ ] Обновить `Version` в `manifest.xml`.
- [ ] Обновить `version` в `package.json`.
- [ ] Обновить `CHANGELOG.md`.
- [ ] Проверить все новые функции и апплеты в документации.
- [ ] Проверить migration/self-healing для изменённой схемы таблиц.

## 2. Автоматические проверки

```bash
npm ci
npm run check
npm run validate:office
```

- [ ] Нет незакоммиченных generated-файлов после `npm run check`.
- [ ] Число функций совпадает в реализации, metadata и docs.
- [ ] Все действия каталога присутствуют в dispatcher.
- [ ] Production build содержит `manifest.xml`, `taskpane.*`, `functions.json`, `assets/`.

## 3. Smoke-тест Excel

### Чистая книга

- [ ] Открытие панели создаёт служебные листы.
- [ ] Листы скрываются после операции.
- [ ] Удаление одной таблицы восстанавливается.

### Функции и апплеты

- [ ] `PROFI.ФИОКРАТКО` работает.
- [ ] Мастер формул вставляет локальную формулу.
- [ ] Шаблон создаётся и повторный запуск не очищает строки.
- [ ] Отчёты и ICS создаются.

### Расписание

- [ ] Импорт внешней книги.
- [ ] Выбор существующего листа.
- [ ] Группа вручную.
- [ ] Группа из выбранной ячейки.
- [ ] Полностью ручной профиль парсера.
- [ ] Алиасы одинаковых фамилий.
- [ ] Потоки включены/выключены.
- [ ] Строгий и совместимый режим.
- [ ] `lock` и `exclude`.
- [ ] Обновление изменённого источника.
- [ ] Восстановление снимка.
- [ ] Все итоговые листы и печать.

## 4. Production

```bash
ADDIN_BASE_URL=https://your-host.example/ npm run build
```

- [ ] Все URL в `dist/manifest.xml` ведут на HTTPS-host.
- [ ] Сервер отдаёт правильные MIME-типы.
- [ ] Отсутствует mixed content.
- [ ] Манифест развёрнут тестовой группе.
- [ ] Выполнена проверка Windows, macOS и web, если они входят в поддержку организации.

## 5. Публикация

- [ ] Создать git tag `vX.Y.Z`.
- [ ] Опубликовать release notes из `CHANGELOG.md`.
- [ ] Приложить `dist/manifest.xml`, если политика проекта разрешает артефакты.
- [ ] Зафиксировать известные ограничения и план отката.
