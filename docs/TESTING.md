# 🧪 Тестирование

## Автоматическая проверка

```bash
npm ci
npm run check
```

Проверяются:

- синтаксис и импорты современного JavaScript;
- 114 функций и 44 апплета;
- автоматическая генерация `functions.json` при сборке;
- browser dashboard и browser preview;
- инициализация task pane без Excel;
- macOS sideload-сценарий без PowerShell;
- парсер, потоки, Round-Robin и конфликты;
- Office.js-манифесты;
- legacy VBA и обязательные процедуры;
- отсутствие x86/x64-зависимых WinAPI-деклараций;
- XLAM `SaveAs` format 55 и XLTM format 53;
- per-user installer и обратимое удаление;
- документация, production-сборка и smoke-тест `dist/`.

Workflow `Windows packaging checks` дополнительно выполняет синтаксический разбор всех `.ps1` и `.psm1` на GitHub-hosted Windows runner.

## Smoke-тест localhost

```bash
npm start
```

Проверьте:

1. `https://localhost:3000/` показывает dashboard;
2. `/health` возвращает `ok: true`, 114 функций и 44 апплета;
3. browser preview содержит заполненные вкладки;
4. стенд `ФИОКРАТКО` возвращает результат;
5. кнопка книги в браузере выводит понятное ограничение;
6. изменение `taskpane.css` вызывает пересборку и перезагрузку preview.

## Ручная приёмка Microsoft 365

1. Загрузите `dist/manifest.xml`.
2. Откройте task pane.
3. Убедитесь, что статус — «Excel подключён».
4. Вставьте формулу из каталога в выбранную ячейку.
5. Запустите «Проверить книгу» в чистой книге.
6. Проверьте автоматическое создание `_PROFI_*` листов.
7. Подключите групповой лист тремя способами определения группы.
8. Намеренно исправьте профиль парсера.
9. Проверьте поток, Round-Robin и конфликт.
10. Закройте и снова откройте книгу.

## Ручная приёмка macOS

```bash
npm run start:mac
```

Проверьте:

- сертификат доверен;
- манифест находится в папке `wef`;
- Excel видит надстройку после перезапуска;
- Custom Functions доступны в поддерживаемой редакции;
- `npm run unsideload:mac` удаляет dev-манифест.

## Реальная сборка Excel

Workflow `Build Legacy Office packages` требует self-hosted runner с метками:

```text
self-hosted, Windows, Excel
```

Он выполняет:

1. `npm ci` и полный `npm run check`;
2. сборку XLAM;
3. сборку XLTM;
4. открытие обоих файлов через Excel COM;
5. проверку RibbonX и VBA-модулей;
6. сборку переносимого ZIP;
7. сборку EXE при наличии Inno Setup;
8. загрузку артефактов.

## Ручная legacy-матрица

| Среда | Проверки |
|---|---|
| Excel 2010 x86 | XLAM, XLTM, UDF, меню, расписание |
| Excel 2013 x86 | установка, шаблон, парсер |
| Excel 2016 x86/x64 | XLAM/XLTM и `manifest-office2016.xml` |
| Excel 2019 x86/x64 | XLAM/XLTM, установщик и обновление |
| Microsoft 365 Windows | современная панель и legacy fallback |

Для каждого клиента проверьте:

- чистую установку и удаление;
- создание новой книги из XLTM;
- автоматическое создание `_PROFI_*` листов;
- группу вручную, из имени листа и из выбранной ячейки;
- ручную коррекцию профиля парсера;
- потоковые занятия и конфликт преподавателя;
- закрытие и повторное открытие книги;
- обновление поверх предыдущей версии.

## Ограничение автоматических тестов

CI не эмулирует пользовательские диалоги Excel, реальное поведение Trust Center, корпоративные политики макросов и все каналы обновления Office. Финальный выпуск требует ручной приёмки по [USE_CASES.md](USE_CASES.md).
