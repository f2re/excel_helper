# 🔐 Приватность и обработка данных

## Кратко

ПрофиПомощник не содержит собственного backend-сервиса и не отправляет содержимое книги в репозиторий или сторонний API. Обработка выполняется в runtime надстройки внутри Excel.

## Где хранятся данные

| Данные | Место хранения |
|---|---|
| Сотрудники, кандидаты, занятия | текущая книга Excel |
| Служебные настройки | `_PROFI_*` листы текущей книги |
| Профили парсера | `tblParserProfiles` |
| Назначения и конфликты | `tblAssignments`, `tblConflicts` |
| История | `tblScheduleVersions`, `tblOperationLog` |
| Исходные групповые листы | в текущей книге, обычно скрытые |
| Код и UI надстройки | локальный или production HTTPS-хостинг |

## Сетевое взаимодействие

- `Office.js` загружается с официального Microsoft CDN, указанного в `taskpane.html`.
- Приложение не содержит fetch/XHR к собственному сервису.
- Production-хостинг отдаёт только статические HTML/JS/CSS/JSON/PNG-файлы.

Организация может разместить production-сборку во внутреннем HTTPS-контуре.

## Персональные данные

Надстройка может обрабатывать ФИО, телефоны, email, кадровые документы и расписания. Ответственность за:

- правомерность обработки;
- доступ к книге;
- срок хранения;
- резервное копирование;
- передачу третьим лицам;
- локальные требования законодательства

остаётся у владельца данных и организации.

## Обезличивание

Апплет A08 создаёт согласованные псевдонимы для выделенных значений. Перед передачей книги:

1. создайте копию;
2. обезличьте необходимые диапазоны;
3. удалите/очистите служебные таблицы, если они содержат исходные ФИО;
4. проверьте скрытые листы;
5. проверьте свойства документа и историю версий.

## Телеметрия

В версии 0.1.0 телеметрия, аналитика использования и автоматическая отправка ошибок отсутствуют.
